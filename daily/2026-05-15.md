## 推理加载顺序

正确顺序：

```python
checkpoint = torch.load(base_checkpoint)
model = MiniGPT(**checkpoint["config"])
model.load_state_dict(checkpoint["model"])
model = replace_qkv_with_lora(model, r=8, alpha=16)
load_lora_adapter(model, adapter_path)
model.eval()
```

为什么不能反过来：

```text
普通 MiniGPT 没有 lora_A/lora_B，adapter 没有地方加载。
adapter 也不包含 token_embedding、FFN、head 等 base 参数，不能单独成为一个模型。
```

## 面试积累

### Q1: LoRA adapter 能不能单独用于推理？

回答草稿：

```text
不能。LoRA adapter 只保存低秩增量矩阵 A/B，不包含完整的 embedding、Transformer、head 等参数。
推理时必须有同结构 base model，再把 adapter 加到对应 Linear 上。
```

### Q2: sampling 时加载 LoRA 的正确顺序是什么？

回答草稿：

```text
先加载 base checkpoint 到普通 MiniGPT，再把目标 Linear 替换成 LoRALinear，
最后加载 LoRA adapter。这样 base 权重和 adapter 权重都有对应位置。
```

### Q3: 如果训练 adapter 时 `r=8`，推理时用 `r=16` 会怎样？

回答草稿：

```text
会 shape 不匹配。adapter checkpoint 里的 A/B shape 是按 r=8 保存的，
推理模型如果创建 r=16 的 LoRA 层，对应权重 shape 不一样，load_state_dict 会报错。
```

## 今日误区

```text
误区：有了 adapter checkpoint 就可以直接 generate。
修正：adapter 只是增量参数，必须配同结构 base checkpoint。
```

```text
误区：strict=False 可以解决 rank 不匹配。
修正：strict=False 允许缺少 key，但 shape 不匹配仍然是结构错误。
```

## 今日一句话

```text
LoRA 推理不是拿 adapter 替代模型，而是把 adapter 贴回同一个 base model 的对应 Linear 上。
```
