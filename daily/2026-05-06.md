## 40 分钟学习

- 来源：
  - `projects/mini-gpt-from-scratch/model.py`
  - `projects/mini-gpt-from-scratch/train.py`
  - `projects/mini-gpt-from-scratch/sample.py`
- 概念：causal self-attention、TransformerBlock、MiniGPT forward、train loss、validation loss、overfitting
- 用自己的话解释：
  - `CausalSelfAttention` 会让每个 token 在不偷看未来的前提下，读取自己和历史 token 的信息。
  - `TransformerBlock` 先用 attention 做跨 token 信息交互，再用 FFN 对每个位置的表示做非线性加工。
  - `MiniGPT` 的输入是 `(batch, seq_len)` 的 token id，输出是 `(batch, seq_len, vocab_size)` 的 logits。
  - 训练时，每个位置都参与 next-token prediction 的 loss。
  - 采样时，每轮只取最后一个位置的 logits，生成 1 个新 token，再拼回上下文。
  - train loss 衡量模型对训练数据的拟合程度，validation loss 衡量模型在未参与训练的数据上的表现。
- 仍然不懂的地方：
  - 需要继续学习 early stopping、regularization、temperature/top-k/top-p 采样策略。

## 面试积累

今天的工作能帮助我回答哪个问题？

### Q1: train loss 和 validation loss 有什么区别？

回答草稿：

```text
train loss 是模型在参与参数更新的数据上的损失，能反映模型对训练集的拟合程度。validation loss 是模型在没有参与训练的数据上的损失，更适合检查模型是否能泛化到没见过的数据。
```

### Q2: 为什么 train loss 很低不一定说明模型真的学会了？

回答草稿：

```text
train loss 很低可能只是说明模型记住了训练集里的具体模式。如果验证集和训练集不同，模型可能表现很差，所以还需要看 validation loss。
```

### Q3: 如果 train loss 下降但 validation loss 上升，说明什么？

回答草稿：

```text
这通常是过拟合信号。模型越来越拟合训练集特定模式，但这些模式不能迁移到验证集，所以训练集表现变好，验证集表现变差。
```

## 今日一句话

```text
validation loss 不是魔法；只有验证数据真的独立于训练数据，它才能帮助判断模型有没有泛化，而不是只背下训练文本。
```
