## 面试积累

### Q1: MiniGPT 里为什么 `qkv` 是 `Linear(128, 384)`？

回答草稿：

```text
MiniGPT 的 hidden dim 是 128，attention 需要同时生成 query、key、value 三组向量，所以输出维度是 3 * dim = 384。
```

### Q2: Day11 为什么只替换 `attn.qkv`？

回答草稿：

```text
这是最小可控实验。`qkv` 是 attention 里的核心线性层，而且 shape 正好对应 Day10 的 Linear(128, 384)。先只替换它，可以清楚观察 LoRA 参数量和冻结逻辑，不把 proj、FFN、head 一起混进来。
```

### Q3: `replace_qkv_with_lora(model)` 做了什么？

回答草稿：

```text
它先冻结整个 MiniGPT 的原始参数，然后遍历每个 TransformerBlock，把 block.attn.qkv 替换成 LoRALinear。LoRALinear 保留原来的 qkv 权重，只额外训练 LoRA A/B。
```

### Q4: 为什么替换 qkv 后 logits shape 不变？

回答草稿：

```text
因为 LoRALinear 的输入输出 shape 和原始 qkv 一样，仍然是 128 -> 384。attention 后续的 split、reshape 和 head 计算看到的张量形状没有变，所以最终 logits shape 仍然是 (batch, seq_len, vocab_size)。
```

## 今日误区

```text
误区：把 qkv 换成 LoRA 后，attention 后面的 shape 会变。
修正：LoRA 分支内部经过 rank 瓶颈，但最终输出仍然回到 out_features，所以外部 shape 不变。
```

```text
误区：只要替换 qkv，其他参数就自然不会训练。
修正：要显式冻结原始 MiniGPT 参数，只保留 LoRA A/B 的 requires_grad=True。
```

## 今日一句话

```text
把 LoRA 接进 MiniGPT 的关键不是改变 attention 的形状，而是在 qkv 旁边加一个可训练的小修正，同时冻结原来的大模型参数。
```
