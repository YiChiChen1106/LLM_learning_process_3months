## 我能解释的概念

### 1. 量化是什么

量化就是把浮点权重压成更少 bit 的离散表示。今天的 toy 版本里，用 `scale = max_abs / 127` 把 float weight 映射到 int8。

### 2. 为什么量化 base model

base model 很大，而且在 LoRA/QLoRA 里是 frozen 的，不需要更新，所以适合压缩存储。

### 3. 为什么 LoRA adapter 不量化

adapter 参数小，但需要梯度和 optimizer 更新，保留较高精度更稳。

### 4. buffer 和 Parameter

- `Parameter` 会进 `model.parameters()`，会被 optimizer 更新。
- `buffer` 会进 `state_dict()`，会跟着模型搬设备，但不会被训练。

### 5. toy 量化公式

```python
q = torch.round(x / scale).clamp(-127, 127).to(torch.int8)
x_hat = q.float() * scale
```

### 6. toy QLoRA forward

```python
out = quantized_base(x) + scaling * lora_B(lora_A(x))
```

其中 `qweight / scale / bias` 是 buffer，`lora_A.weight / lora_B.weight` 是 Parameter。

### 7. toy QLoRA merge

```python
base_weight_hat = qweight.float() * scale
delta_weight = lora_B.weight @ lora_A.weight
merged_weight = base_weight_hat + scaling * delta_weight
```

merge 后模型变回普通 `nn.Linear`，适合推理，不再保留 adapter 训练结构。

## 今天的一个误区

量化不是把模型删掉，而是用更粗的数值表示同一批权重。

## 今天的结论

QLoRA 可以先记成一句话：

```text
冻结的大模型压缩存，LoRA 小补丁保持可训练。
```
