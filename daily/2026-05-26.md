## 我能解释的概念

### 1. adapter 不能单独 generate

LoRA adapter 只包含 `lora_A/lora_B`，没有 token embedding、position embedding、FFN、head、LayerNorm 和 base qkv 权重，所以它不是完整模型。

### 2. base + adapter 的加载顺序

```python
model = MiniGPT(**checkpoint["config"])
model.load_state_dict(checkpoint["model"])
model = replace_qkv_with_lora(model, r=8, alpha=16)
load_lora_adapter(model, adapter_path)
```

必须先加载普通 base 权重，再把 qkv 替换成 LoRA，最后加载 adapter。

### 3. merge 后不再需要 adapter

merge 会把 LoRA 增量合进普通 Linear：

```python
delta_weight = lora_B.weight @ lora_A.weight
merged_weight = base_weight + scaling * delta_weight
```

merge 后的 checkpoint 是完整普通 MiniGPT checkpoint，可以直接 sample。

### 4. 为什么比较 logits 比较严谨

文本生成会受 greedy、temperature、top-k、top-p、seed 等采样策略影响。logits 是同一个 input 下模型 forward 的直接输出，更适合验证 `base + adapter` 和 `merged checkpoint` 是否等价。

## 今天的误区提醒

- adapter 改变输出，不代表 adapter 本身是完整模型。
- merged checkpoint 和 adapter checkpoint 不是一类文件。
- 文本一样可以作为直观信号，但 logits 一致才是更严格的等价验证。
- merge 后 `state_dict()` 里不应该再有 `lora_A/lora_B`。
