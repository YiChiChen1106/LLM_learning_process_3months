## 面试积累

### Q1: `max_steps` 控制什么？它和模型结构有没有关系？

回答草稿：

```text
max_steps 控制训练循环最多执行多少次参数更新，它属于训练流程超参数，不改变模型结构。模型结构由层数、隐藏维度、注意力头数、block_size 等决定。
```

### Q2: `eval_interval` 会不会改变模型每一步的参数更新？

回答草稿：

```text
不会。eval_interval 只控制隔多少 step 运行一次评估逻辑，比如计算 train_loss、val_loss、保存 best checkpoint、检查 early stopping。参数更新仍然发生在每个训练 step。
```

### Q3: 为什么 patience 的实际等待时间会受到 eval_interval 影响？

回答草稿：

```text
patience 统计的是连续多少次评估没有改善，而 eval_interval 决定两次评估之间隔多少训练 step。所以实际等待的训练步数大约等于 patience * eval_interval。
```

### Q4: `max_steps` 和 early stopping 谁决定训练结束？

回答草稿：

```text
max_steps 和 early stopping 都可能决定训练结束。max_steps 是最多训练多少步的硬上限；early stopping 是当 validation loss 连续多次不改善时提前停止。哪个条件先满足，训练就在哪停止。
```

### Q5: learning rate 太大可能会带来什么问题？

回答草稿：

```text
learning rate 太大时，每次参数更新步子过大，可能导致训练不稳定。在这个实验里，它让 train loss 很快降到很低，但 validation loss 大幅升高，说明模型更快地过拟合训练集，泛化表现更差。
```

## 今日一句话

```text
max_steps 管最多训练多久，eval_interval 管多久检查一次，patience 管能忍几次没改善，learning_rate 管每次更新迈多大步；这些控制训练流程，不改变模型结构。
```
