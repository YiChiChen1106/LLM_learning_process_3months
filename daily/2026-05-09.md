## 面试积累

### Q1: 为什么最后一步训练出来的模型不一定是最好的模型？

回答草稿：

```text
最后一步模型通常是在训练集上继续优化后的结果，它的 train loss 可能更低，但如果模型已经过拟合，validation loss 可能已经上升。所以最后一步模型不一定泛化最好，真正应该关注的是 validation loss 最低时的模型。
```

### Q2: 为什么 `best_val_loss` 一开始要设成 `float("inf")`？

回答草稿：

```text
一开始还没有历史最好 validation loss，所以先把 best_val_loss 设成正无穷。这样第一次真实的 val_loss 一定会小于它，保证第一次评估能保存一个初始 best checkpoint。
```

### Q3: early stopping 应该看 train loss 还是 validation loss？

回答草稿：

```text
early stopping 应该看 validation loss，因为它反映模型在未参与训练的数据上的表现。train loss 往往会随着训练继续下降，但这可能只是模型越来越会拟合训练集。只有 validation loss 能帮助判断模型的泛化表现是否还在改善。
```

### Q4: early stopping 触发时，best checkpoint 是停止那一步的模型吗？

回答草稿：

```text
不一定。early stopping 的停止点只是训练被提前终止的位置；best checkpoint 保存的是整个训练历史中 validation loss 最低的那一步模型。
```

## 今日一句话

```text
best checkpoint 负责保存验证集表现最好的模型，early stopping 负责在验证集连续不改善时停止训练；它们都是用 validation loss 防止模型只会背训练集。
```
