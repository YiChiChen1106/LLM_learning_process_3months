## 为什么 eval 不能混进 train

今天先明确了一个核心概念：

```text
train 数据：给模型看，用来更新 adapter 参数
eval 数据：不给模型看，只用来考试
```

如果 eval prompt 原句出现在 train 里，就会发生 data leakage。

这就像：

```text
考试前把考卷和答案发给学生背
```

最后分数高，不代表真的理解，只能说明它见过这道题。

昨天 V5 把固定评测 prompt 原句放进训练集，是为了做纠错实验，可以接受。

但正式评测不能这样做。

更合理的方式是：

```text
train:
LoRA 是参数高效微调方法，不是预训练方法。

eval:
有人说 LoRA 属于 pretraining，这种说法对吗？
```

同一个知识点，换一种问法，才能测试模型是不是学到了概念。


也就是 eval prompt 没有和 train instruction 完全重复。

### curated_train 的作用

`curated_train.jsonl` 用来教模型标准答案。

特点：

- 清楚
- 短
- 稳定
- 不自相矛盾
- 不写模糊机制
- 不把错误说法写进 response

覆盖主题：

- LoRA 是什么
- LoRA vs full fine-tuning
- QLoRA vs LoRA
- adapter checkpoint
- adapter_model.safetensors
- target_modules
- labels=-100
- attention_mask
- collator
- gradient accumulation
- base 输出对照

### curated_eval 的作用

`curated_eval.jsonl` 用来考试。

特点：

- 不复读 train 的原句
- 用不同问法考同一个概念
- 每条有 `expected` 字段，方便人工判断

例子：

```json
{"id": "adapter_only_rephrase", "prompt": "我只有 adapter_model.safetensors，没有下载 base model，可以直接让它聊天吗？", "expected": "应答出不能，adapter 不是完整模型，必须和对应 base model 一起加载。"}
```

这比直接问：

```text
adapter checkpoint 为什么不能单独 generate？
```

更像真实用户会问的问题。

## 错误分类表

新增：

```text
projects/llm-lora-finetuning/error_taxonomy.md
```

今天把错误分成六类：

```text
factual_error
concept_confusion
hallucinated_mechanism
incomplete_answer
format_issue
off_topic
```

### factual_error

事实错误。

例子：

```text
说 QLoRA 量化 adapter
说 adapter_model.safetensors 包含完整 base model
说 LoRA 更新所有 base weights
```

### concept_confusion

概念混淆。

例子：

```text
把 LoRA 和 pretraining 混在一起
把 LoRA 和模型压缩混在一起
把 attention_mask 和 labels=-100 混在一起
```

### hallucinated_mechanism

幻觉机制。

例子：

```text
编造 base_model.ckpt 转换流程
说 LoRA 会删除不相关参数
说 adapter checkpoint 可以单独作为模型加载
```

### incomplete_answer

答案部分正确，但漏掉关键点。

例子：

```text
只说 adapter 不能单独 generate
但没说必须加载 base model
```

### format_issue

格式问题。

例子：

```text
没有先说结论
回答太长
表达不清楚
```

### off_topic

答非所问。

例子：

```text
问 adapter checkpoint，答成 KV cache
```

## 今天的核心结论

今天最重要的一句话：

```text
训练只是第一步，评测流程决定我们知不知道模型到底有没有变好。
```

更具体地说：

```text
1. eval 不能混进 train。
2. train 负责教，eval 负责考。
3. loss 不能代替人工评估。
4. 固定评测集能暴露模型是否真的泛化。
5. 错误分类能让改数据更有方向。
6. 标注汇总能把“感觉模型变好了”变成可比较的数字和错误分布。
```
