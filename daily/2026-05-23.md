## 核心公式

对于：

```python
qkv = nn.Linear(128, 384)
```

PyTorch 里的权重 shape 是：

```text
qkv.weight: (384, 128)
```

LoRA 的两个权重是：

```text
lora_A.weight: (8, 128)
lora_B.weight: (384, 8)
```

所以：

```text
lora_B.weight @ lora_A.weight = (384, 128)
```

这个 shape 正好可以加到原始 `qkv.weight` 上：

```python
merged_weight = base_weight + scaling * (lora_B.weight @ lora_A.weight)
```

## 面试积累

### Q1: LoRA merge 的公式是什么？

回答草稿：

```text
LoRA 的输出等价于在 base weight 上增加一个低秩增量。
对 PyTorch Linear 来说，delta_W = scaling * (lora_B.weight @ lora_A.weight)，
所以 W_merged = W_base + delta_W。
```

### Q2: merge 后为什么模型里没有 adapter 参数？

回答草稿：

```text
因为 adapter 的 A/B 已经被乘成 delta_W，并加进普通 Linear.weight。
merge 后 qkv 从 LoRALinear 替换回 nn.Linear，state_dict 里自然不再有 lora_A/lora_B。
```

### Q3: merged checkpoint 和 adapter checkpoint 有什么区别？

回答草稿：

```text
adapter checkpoint 只保存 lora_A/lora_B，必须配 base checkpoint 使用。
merged checkpoint 保存完整普通 MiniGPT 的 state_dict、config、stoi、itos，
采样时可以直接用 --checkpoint merged.pt。
```
