## 作答记录

一次学习记录最少保存：

```json
{
  "prompt_id": "concept_qlora_memory",
  "prompt": "QLoRA 为什么更省显存，是因为 adapter 参数更少吗？",
  "expected": "应答出不是 adapter 更少，而是 frozen base weights 4bit 量化加载。",
  "user_answer": "因为 adapter 参数更少",
  "grade": "wrong",
  "error_types": ["concept_confusion"],
  "feedback": "QLoRA 更省显存主要不是因为 adapter 参数更少，而是 frozen base weights 被 4bit 量化加载。",
  "review_tip": "复习 QLoRA 省的是 frozen base weights 的存储显存，不是 LoRA adapter 参数量。"
}
```

这里最重要的是：

- `prompt_id` 把用户答案和题库里的标准要点关联起来。
- `grade` 记录答题结果。
- `error_types` 说明错在哪里。
- `feedback` 给本次回答的解释。
- `review_tip` 把错题转成复习点。

## 自动粗判

今天没有用模型判分，而是先做了透明的规则判分。

规则文件：

```text
data/study_grading_rules.jsonl
```

每条规则包含：

```text
required: 必须命中的关键点
forbidden: 不能出现的错误说法
feedback_correct: 答对时的反馈
review_tip: 复习建议
```

以 `concept_qlora_memory` 为例：

```text
required:
- frozen base weights
- 4bit / 量化

forbidden:
- adapter 参数更少
- 可训练参数更少
```

判分顺序：

```text
先看 forbidden
-> 命中错误说法就判 wrong
-> 再看 required 命中情况
-> 全命中 correct
-> 命中一部分 partial
-> 全没命中 wrong
```

这不是语义理解模型，而是一个可控的规则判分器。它的优点是透明、容易调试；缺点是比较死板，需要不断补充同义表达。

## 错题复盘

今天还加了复盘统计：

```powershell
.\.venv\Scripts\python.exe projects\llm-lora-finetuning\study_assistant.py --review --review-md outputs\study_review.md
```

它会统计：

- 总作答数
- 已判分数
- 需要复习数：`partial + wrong`
- accuracy
- `grade_counts`
- `error_type_counts`
- `category_counts`
- `needs_review_by_category`
- `weak_prompt_counts`
- `review_tips`

这样学习助手不只是告诉我“这一题错了”，还可以告诉我“最近经常错在哪类问题”。

## 推荐下一题

今天最后补了下一题推荐：

```powershell
.\.venv\Scripts\python.exe projects\llm-lora-finetuning\study_assistant.py --recommend-next
```

推荐策略是确定性的：

```text
1. 找 partial/wrong 最多的题目类别。
2. 优先推荐这个类别里还没答过的题。
3. 如果这个类别都答过了，就推荐错得最多的旧题复习。
4. 如果没有明显错题，就推荐第一道未作答题。
```

例如如果 `dataflow_attention_mask` 答错，系统会优先推荐同属 `dataflow` 的题，比如：

```text
dataflow_loss_ignore_prompt
```

这一步让学习助手从“题库播放器”变成了“会根据薄弱点安排下一题”的系统。
