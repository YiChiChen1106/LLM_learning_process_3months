这个脚本不是为了训练出一个有实际用途的大模型，而是为了看清楚所有模型训练背后的最小骨架：

```text
输入 -> 模型预测 -> 计算 loss -> backward 算梯度 -> optimizer 更新参数 -> loss 下降
```

可以把它想象成一个新手调音师在调 4 个旋钮。模型一开始乱猜，loss 很高；每次训练都会根据错误程度计算每个旋钮该往哪个方向调，然后优化器真正调整旋钮。训练 10 步后，loss 从 `1.5130` 降到 `0.4546`，说明模型确实学到了一点东西。

## 脚本在做什么

```python
x = torch.randn(8, 4)
y = torch.randn(8, 1)
linear = torch.nn.Linear(4, 1)
optimizer = torch.optim.SGD(linear.parameters(), lr=0.1)
```

- `x` 是输入，形状是 `(8, 4)`：8 条样本，每条样本 4 个特征。
- `y` 是目标答案，形状是 `(8, 1)`：8 条样本，每条对应 1 个目标值。
- `linear` 是一个线性模型，把 4 个输入数字变成 1 个输出数字。
- `optimizer` 是 SGD 优化器，负责根据梯度更新模型参数。

每一步训练做的是：

```python
pred = linear(x)
loss = torch.nn.functional.mse_loss(pred, y)

optimizer.zero_grad(set_to_none=True)
loss.backward()
optimizer.step()
```

- `pred = linear(x)`：模型先猜答案。
- `loss = mse_loss(pred, y)`：计算模型猜得离真实答案有多远。
- `optimizer.zero_grad()`：清空上一轮留下的梯度。
- `loss.backward()`：反向求导，计算每个参数对 loss 的影响。
- `optimizer.step()`：结合学习率，真正更新参数。

## `.grad` 是什么

`.grad` 是每个可训练参数旁边存放梯度的地方。

比如：

```python
linear.weight
```

是当前权重，也就是当前旋钮的位置。

执行：

```python
loss.backward()
```

之后，PyTorch 会把权重对应的梯度存到：

```python
linear.weight.grad
```

可以这样理解：

```text
linear.weight      = 当前旋钮位置
linear.weight.grad = 这次 backward 算出来的调整建议
```

但 `.grad` 不是最终改变量。最终怎么改，还要看学习率和优化器规则。真正修改参数的是：

```python
optimizer.step()
```

## 计算图是什么

计算图是 PyTorch 在前向计算时偷偷画下来的路线图。

脚本里发生了：

```text
x, weight, bias
      ↓
   linear
      ↓
    pred
      ↓
  mse_loss
      ↓
    loss
```

当执行：

```python
loss.backward()
```

PyTorch 会从 `loss` 开始沿着这张图反着走：

```text
loss -> mse_loss -> pred -> linear -> weight, bias
```

一路计算：

```text
loss 对 weight 的影响是多少？
loss 对 bias 的影响是多少？
```

最后把结果写进：

```python
linear.weight.grad
linear.bias.grad
```

一句话记忆：

```text
前向传播时，PyTorch 画地图；backward 时，PyTorch 沿地图倒着走，把每个参数的梯度写进 .grad。
```

## 梯度形状是什么

梯度形状就是梯度这个张量的尺寸。

梯度和参数是一一对应的，所以参数是什么形状，它对应的梯度通常也是什么形状。

脚本里：

```python
linear = torch.nn.Linear(4, 1)
```

PyTorch 的线性层权重形状规则是：

```text
(out_features, in_features)
```

所以：

```text
linear.weight      = (1, 4)
linear.weight.grad = (1, 4)
```

可以想象成：

```text
权重： [ w1  w2  w3  w4 ]
梯度： [ dw1 dw2 dw3 dw4 ]
```

权重有 4 个旋钮，梯度就有 4 条对应的调整建议。

## 三个问题的复盘

### 1. `loss.backward()` 做了什么？

我的初始理解：

```text
它通过反向求导，结合学习率，算出来各个权重的改变量。
```

纠正后：

```text
loss.backward() 会从 loss 出发，沿计算图反向求导，计算 loss 对 linear.weight 和 linear.bias 的梯度，并把结果存到它们的 .grad 属性里。它本身不会更新参数，也不会使用学习率。真正结合学习率更新参数的是 optimizer.step()。
```

记忆句：

```text
backward 写建议，step 动旋钮。
```

### 2. 为什么需要 `optimizer.zero_grad(set_to_none=True)`？

我的回答：

```text
原来的 .grad 有上次反向传播的结果，需要删除，防止误导这次的权重调整。
```

这个理解是对的。更准确的表达是：

```text
PyTorch 默认会把每次 backward 得到的梯度累加到参数的 .grad 里。如果不在每轮训练前清零，上一轮的梯度会和这一轮混在一起，导致 optimizer.step() 用错误的梯度更新参数。所以每个 step 前要调用 optimizer.zero_grad()。
```

`set_to_none=True` 的意思：

```text
把 .grad 设成 None，而不是填成 0。这样通常更省内存，也能让下一次 backward 重新创建梯度。
```

### 3. 脚本里出现了哪些 tensor shape？

正确答案：

```text
x:                  (8, 4)
y:                  (8, 1)
pred:               (8, 1)
linear.weight:      (1, 4)
linear.bias:        (1,)
linear.weight.grad: (1, 4)
```

容易错的地方：

```text
Linear(4, 1) 的 weight 不是 (4, 1)，而是 (1, 4)。
```

原因：

```text
PyTorch 的 Linear(in_features, out_features) 权重形状是 (out_features, in_features)。
```

## 今日能讲清楚的一句话

PyTorch 的最小训练循环是：模型先根据输入做预测，loss 衡量预测和目标的差距，`zero_grad()` 清掉上一轮梯度，`backward()` 沿计算图反向求导并把梯度写进 `.grad`，最后 `optimizer.step()` 结合学习率更新参数；如果 loss 逐步下降，就说明模型正在朝更好的参数方向移动。
