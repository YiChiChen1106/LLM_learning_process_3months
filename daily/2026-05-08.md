## 面试积累

### Q1: 为什么 `torch.multinomial(probs, num_samples=1)` 会导致同样输入生成不同文本？

回答草稿：

```text
torch.multinomial 不是取概率最大的 token，而是按照 softmax 得到的概率分布进行随机抽样。高概率 token 更容易被选中，但低概率 token 也有机会被选中，所以同一个 prompt 在相同参数下多次生成，结果也可能不同。
```

### Q2: `torch.manual_seed(42)` 的作用是什么？

回答草稿：

```text
torch.manual_seed(42) 用来固定 PyTorch 的随机数生成过程，使包含随机操作的代码在相同条件下得到可复现结果。它不会改变模型参数，也不会提升模型能力，只是让随机采样结果稳定复现。
```

### Q3: greedy decoding 和 `torch.multinomial` 最大的区别是什么？

回答草稿：

```text
greedy decoding 每一步都选择当前概率最大的 token，是确定性解码；torch.multinomial 是按照概率分布随机抽样，高概率 token 更容易被选中，但低概率 token 也可能被选中，所以生成结果更有随机性。
```

### Q4: 为什么 greedy 生成更稳定，但不一定更好？

回答草稿：

```text
greedy decoding 更稳定，因为每一步都选择当前概率最高的 token，几乎没有随机性。但它不一定更好，因为局部概率最高的 token 不一定能组成全局最自然的文本，而且缺少随机性，容易产生重复、单调或过早走进固定模式。
```

### Q5: `seed` 和 `greedy` 都能让输出更稳定，它们的区别是什么？

回答草稿：

```text
seed 是固定随机数生成过程，让随机采样可复现，但它仍然是在概率分布里抽样；greedy 是直接选择概率最大的 token，基本不做随机抽样。
```

### Q6: `model.eval()` 和 `torch.no_grad()` 的区别是什么？

回答草稿：

```text
model.eval() 是切换模型内部模块的行为，比如关闭 dropout、让 batchnorm 使用固定统计量；torch.no_grad() 是关闭梯度追踪，让 PyTorch 不记录计算图，也不保存反向传播需要的中间结果。两者解决的问题不同，推理时通常会一起使用。
```

### Q7: 为什么 sample.py 推理生成时不需要记录计算图？

回答草稿：

```text
sample.py 的生成阶段只是用已经训练好的权重做前向推理，根据 logits 选择下一个 token，并不会计算 loss、执行 backward 或更新参数。计算图主要服务于反向传播，所以推理阶段不需要记录计算图。
```

### Q8: 为什么 `torch.no_grad()` 能节省内存？

回答草稿：

```text
torch.no_grad() 会关闭梯度追踪，因此 PyTorch 不需要保存反向传播所需的中间激活值和计算图信息。少保存这些中间结果，就能减少内存或显存占用。
```
