## 为什么 smoke 成功不等于效果好

这次训练条件很小：

```text
训练数据：3 条
epoch：1
global step：1
trainable params：1,081,344
```

adapter 有 100 多万个可训练参数，但只看了 3 条样本，只更新 1 次。

所以：

```text
工程链路：成功
模型效果：还不能评价
```

smoke 成功代表：

```text
数据能读；
tokenize / labels / collator 能工作；
Trainer 能算 loss 和 backward；
LoRA adapter 能保存；
base + adapter 能重新加载并 generate。
```

不代表：

```text
模型已经学会新知识；
回答质量已经提高；
微调已经有效。
```

## LoRA 和 QLoRA 的 trainable params 为什么一样

LoRA 和 QLoRA 训练的可训练部分基本一样：

```text
lora_A
lora_B
```

所以两者都是：

```text
trainable params: 1,081,344
adapter_model.safetensors: 4.2M
```

区别在 base：

```text
LoRA:
base model 用 bf16/fp16/fp32 加载。

QLoRA:
frozen base model 用 4bit 加载。
```

所以 QLoRA 省的是：

```text
frozen base model 的显存
```

不是：

```text
adapter 参数量
```

## QLoRA adapter generate smoke

最后验证 4bit base + QLoRA adapter 推理。

关键代码：

```python
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
)

base_model = AutoModelForCausalLM.from_pretrained(
    base_model_name,
    dtype=torch.bfloat16,
    device_map="auto",
    quantization_config=bnb_config,
    trust_remote_code=True,
)

model = PeftModel.from_pretrained(base_model, adapter_path)
```

结果：

```text
model_device cuda:1
cuda True
adapter_path runs/qwen-smoke-qlora
is_loaded_in_4bit True
```

说明这条真实 QLoRA 推理路径通了：

```text
Qwen2.5-0.5B base
-> BitsAndBytesConfig(load_in_4bit=True)
-> 4bit quantized base loaded on GPU
-> PeftModel.from_pretrained(... qlora adapter ...)
-> generate
```

## 重要结论

LoRA adapter checkpoint 不能单独推理。

它必须挂到对应 base model 上：

```text
普通 LoRA 推理:
bf16/fp16/fp32 base + adapter

QLoRA 推理:
4bit base + adapter
```

如果想保持 QLoRA 的省显存优势，推理时也要：

```text
4bit 加载 base model
再挂 adapter
```

不能只加载 adapter。

验证：

```text

LoRA adapter on quantized base；
4bit base + adapter 推理。
```

### Q7: 为什么 LoRA 和 QLoRA 的 trainable params 一样？

因为两者训练的都是 LoRA adapter。

区别是 frozen base 是否 4bit 量化。

### Q8: QLoRA adapter 能不能单独 generate？

不能。

adapter 不包含完整 base model。

它必须挂到 base model 上。
