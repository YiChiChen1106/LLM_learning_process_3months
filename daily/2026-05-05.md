## 40 分钟学习

- 来源：`projects/pytorch-llm-basics/tiny_lm.py`
- 概念：next-token prediction、`block_size`、`dim`、train loss、validation loss、overfitting
- 用自己的话解释：
  - `CORPUS` 是训练语料库。
  - `chars` 是从语料中提取、去重、排序后的字符表。
  - `stoi` 把字符映射成 id，`itos` 把 id 映射回字符。
  - `data` 是整个 `CORPUS` 被转换成 token id 后的一维张量。
  - `starts` 是在 `data` 里随机找 batch size 个起点。
  - `x` 是每个起点开始、长度为 `block_size` 的 token 序列。
  - `y` 是 `x` 往后错一位的标准答案。
  - 模型接收 `x` 输出 `logits`，再和 `y` 做 cross entropy 得到 loss。
- 仍然不懂的地方：
  - 需要继续学习 GPT 里的 causal self-attention，理解 token 之间到底怎么发生信息交互。

  - 原因是当前语料太短，无法切出长度为 64 的 `x`，还要再往后一位切出 `y`。
- 学到了什么：
  - `block_size` 只是每次训练片段长度，不代表模型真的能理解这么长的上下文。
  - `TinyBigramLM` 的结构只有 `Embedding -> Linear`，不同 token 位置之间没有 attention、RNN 或其他信息交互。
  - `dim` 增大后模型容量变大，loss 可能更低，但生成文本仍然会重复，因为结构限制没有改变。

## 面试积累

今天的工作能帮助我回答哪个问题？

### Q1: 为什么 train loss 下降，不一定说明模型真的变聪明了？

回答草稿：

```text
train loss 下降只说明模型在训练集上越来越会做题，可能只是把训练数据里的模式甚至答案背熟了。就像模拟卷刷多了，模拟卷错得越来越少，但如果没有真正掌握规律，遇到真卷里的新题还是可能不会做。这就是过拟合。
```

### Q2: validation loss 是用来检查什么的？

回答草稿：

```text
validation loss 是用没有参与训练的数据计算出来的损失，用来检查模型对没见过的数据表现如何。因为这些数据没有被模型拿来更新参数，所以它比 train loss 更能反映模型是否学到了可迁移的规律，也就是泛化能力。
```

### Q3: 如果 train loss 一直下降，但 validation loss 先下降后上升，说明什么？

回答草稿：

```text
这通常说明过拟合了。模型越来越会拟合训练集，但对没见过的验证集表现开始变差。
```
