## 我能解释的概念

### 1. full fine-tuning 和 LoRA 的区别

full fine-tuning 会更新模型里的所有参数。LoRA 冻结原模型参数，只训练额外加上的 `lora_A` / `lora_B`。

### 2. 为什么 LoRA 主要省的是 trainable parameters

LoRA 会额外增加一小段低秩分支，所以 `total parameters` 往往会变多；但真正参与更新的只有 LoRA 参数，所以 `trainable parameters` 会大幅下降。

### 3. adapter、base checkpoint、merged checkpoint

- `base checkpoint`：完整 MiniGPT，可以单独 sample。
- `LoRA adapter`：只保存 `lora_A/lora_B`，不能单独 sample。
- `merged checkpoint`：把 LoRA 增量合进 base weight 后的完整模型，也可以单独 sample。

### 4. merge 公式

```python
delta_weight = lora_B.weight @ lora_A.weight
merged_weight = base_weight + scaling * delta_weight
```

### 5. 为什么比较 logits 更严谨

文本生成会受 greedy、temperature、top-k、top-p 和随机采样影响。logits 是模型 forward 的直接输出，更适合验证 `base + adapter` 和 `merged checkpoint` 是否等价。

## 今天的一个误区

冻结参数不等于不参与 forward。`requires_grad=False` 只表示不更新、不求导，不表示这部分参数会从计算图里消失。

## 今天的结论

LoRA 的本质可以记成一句话：

```text
冻结大模型，训练小补丁；merge 后补丁并回主模型，直接推理。
```
