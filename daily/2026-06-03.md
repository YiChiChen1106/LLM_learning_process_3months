## Trainer 到模型 forward

在 `transformers/trainer.py` 里，`Trainer.compute_loss` 会把 batch 喂给模型：

```python
outputs = model(**inputs)
```

所以当前训练数据里的这些字段：

```text
input_ids
attention_mask
labels
```

会直接进入模型 forward。

如果没有自定义 loss，大多数 Causal LM 的 loss 是模型 forward 内部算出来的。

## Qwen2ForCausalLM 的 loss 入口

在 `transformers/models/qwen2/modeling_qwen2.py` 里，`Qwen2ForCausalLM.forward` 会先算 logits：

```python
logits = self.lm_head(hidden_states[:, slice_indices, :])
```

如果传入了 labels，就调用统一 loss：

```python
loss = None
if labels is not None:
    loss = self.loss_function(
        logits=logits,
        labels=labels,
        vocab_size=self.config.vocab_size,
        **kwargs,
    )
```

这说明我们在 `train_lora.py` 里构造的 labels mask，最终会进入这里。

## ForCausalLMLoss 里的 shift

在 `transformers/loss/loss_utils.py` 里，本地版本的核心逻辑是：

```python
def ForCausalLMLoss(
    logits,
    labels,
    vocab_size: int,
    num_items_in_batch: torch.Tensor | None = None,
    ignore_index: int = -100,
    shift_labels: torch.Tensor | None = None,
    **kwargs,
) -> torch.Tensor:
    logits = logits.float()

    if shift_labels is None:
        # Shift so that tokens < n predict n
        labels = nn.functional.pad(labels, (0, 1), value=ignore_index)
        shift_labels = labels[..., 1:].contiguous()

    logits = logits.view(-1, vocab_size)
    shift_labels = shift_labels.view(-1)
    shift_labels = shift_labels.to(logits.device)
    loss = fixed_cross_entropy(
        logits,
        shift_labels,
        num_items_in_batch,
        ignore_index,
        **kwargs,
    )
    return loss
```

以前常见写法是：

```python
shift_logits = logits[..., :-1, :]
shift_labels = labels[..., 1:]
```

本地 Transformers 5.7.0 的写法稍有不同：它没有显式裁掉最后一个 logits，而是先在 labels 末尾补一个 `-100`，再右移 labels：

```python
labels = pad(labels, (0, 1), value=-100)
shift_labels = labels[..., 1:]
```

效果等价：

```text
第 t 个位置的 logits -> 预测第 t+1 个 token
最后一个位置 -> 对应 -100，被忽略
```

## cross_entropy 里的 ignore_index

同一个文件里的 `fixed_cross_entropy`：

```python
def fixed_cross_entropy(
    source: torch.Tensor,
    target: torch.Tensor,
    num_items_in_batch: torch.Tensor | None = None,
    ignore_index: int = -100,
    **kwargs,
) -> torch.Tensor:
    reduction = "sum" if num_items_in_batch is not None else "mean"
    loss = nn.functional.cross_entropy(
        source,
        target,
        ignore_index=ignore_index,
        reduction=reduction,
    )
```

所以 `-100` 真正被忽略的位置是：

```python
cross_entropy(ignore_index=-100)
```

它不是 tokenizer 忽略的，也不是 attention mask 自动忽略的。

## flatten 的意义

源码里：

```python
logits = logits.view(-1, vocab_size)
shift_labels = shift_labels.view(-1)
```

如果：

```text
logits shape       = [2, 5, 50000]
shift_labels shape = [2, 5]
```

flatten 后：

```text
logits       -> [10, 50000]
shift_labels -> [10]
```

含义是：

```text
2 个 batch * 5 个位置 = 10 个分类任务
每个任务从 vocab_size 个 token 里选正确答案
```

其中 label 是 `-100` 的任务会被 `ignore_index=-100` 跳过。

## 和 Day24 代码对齐

Day24 的 `train_lora.py` 里生成 labels：

```python
encoded["labels"] = [
    token_id if mask == 1 and i >= prompt_len else -100
    for i, (token_id, mask) in enumerate(
        zip(encoded["input_ids"], encoded["attention_mask"])
    )
]
```

它的含义是：

```text
prompt / user / template -> -100
assistant response       -> token id
padding                  -> -100
```

然后 Transformers 做：

```text
labels 右移
logits / labels flatten
cross_entropy(ignore_index=-100)
```

最终实际训练的是：

```text
看到 prompt，预测 assistant 第一个 token
看到 assistant 前文，预测 assistant 下一个 token
padding 和 prompt 位置不算 loss
```

## toy loss 实验

今天跑了一个最小 PyTorch 实验：

```python
import torch
import torch.nn.functional as F

torch.manual_seed(0)

batch = 1
seq_len = 5
vocab_size = 4

labels = torch.tensor([[-100, -100, 1, 2, -100]])
logits = torch.randn(batch, seq_len, vocab_size)

padded_labels = F.pad(labels, (0, 1), value=-100)
shift_labels = padded_labels[..., 1:].contiguous()
flat_logits = logits.view(-1, vocab_size)
flat_labels = shift_labels.view(-1)

loss = F.cross_entropy(flat_logits, flat_labels, ignore_index=-100)

active = flat_labels != -100
manual_loss = F.cross_entropy(flat_logits[active], flat_labels[active])
```

输出结果：

```text
labels        = [[-100, -100, 1, 2, -100]]
shift_labels  = [[-100, 1, 2, -100, -100]]
flat_labels   = [-100, 1, 2, -100, -100]
active labels = [1, 2]
loss          = 2.200226
manual_loss   = 2.200226
same?         = True
```

这证明：

```text
cross_entropy(全部位置, ignore_index=-100)
=
cross_entropy(只取非 -100 的位置)
```

真正参与 loss 的 label 只有：

```text
[1, 2]
```

## attention_mask 和 labels=-100 的区别

今天最后澄清了一个问题：这件事和 `attention_mask` 有关系，但 `attention_mask` 不是最终忽略 loss 的东西。

两者分工：

```text
attention_mask = 0
-> 告诉模型 attention 时别看 padding 位置

labels = -100
-> 告诉 loss 这个位置不要算训练目标
```

在我们的代码里，`attention_mask` 用来判断哪里是 padding：

```python
token_id if mask == 1 and i >= prompt_len else -100
```

但真正让这个位置不参与 loss 的，是 `labels=-100` 和：

```python
cross_entropy(ignore_index=-100)
```

一句话：

```text
attention_mask 管输入侧；
labels=-100 管监督信号侧。
```

## 今天能回答的问题

### Q1: `-100` 最终在哪里被忽略？

在 `cross_entropy(ignore_index=-100)` 里被忽略。

### Q2: 本地 Transformers 5.7.0 怎么做 Causal LM shift？

它先给 labels 末尾 pad 一个 `-100`，再取：

```python
shift_labels = labels[..., 1:]
```

等价于让第 t 个位置预测第 t+1 个 token。

### Q3: flatten 后 logits 和 labels 分别是什么 shape？

如果原来是：

```text
logits       = [batch, seq_len, vocab_size]
shift_labels = [batch, seq_len]
```

flatten 后是：

```text
logits       = [batch * seq_len, vocab_size]
shift_labels = [batch * seq_len]
```

### Q4: `attention_mask=0` 和 `labels=-100` 有什么区别？

`attention_mask=0` 影响模型 attention；`labels=-100` 影响 loss 是否计算。

## 今天的结论

```text
数据阶段：train_lora.py 把 prompt/padding 标成 -100
模型阶段：Causal LM loss 把 labels shift
loss 阶段：cross_entropy 忽略 -100
最终只训练 assistant response token
```

再压缩成一句话：

```text
attention_mask 管“模型能不能看”；
labels=-100 管“这个位置算不算 loss”；
shift 管“第 t 个位置预测第 t+1 个 token”。
```
