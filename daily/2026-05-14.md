## 参数流推导

### adapter state_dict 筛选

```python
def get_lora_state_dict(model):
    return {
        name: tensor
        for name, tensor in model.state_dict().items()
        if "lora_" in name
    }
```

关键区别：

```text
训练参数筛选：看 parameter.requires_grad
保存 adapter 筛选：看 state_dict key name
```

### adapter 参数量

```text
单个 qkv:
lora_A: 8 * 128 = 1024
lora_B: 384 * 8 = 3072
单层合计: 4096

4 个 block:
4096 * 4 = 16384
```

### 加载顺序

```python
model = MiniGPT(...)
model = replace_qkv_with_lora(model, r=8, alpha=16)
load_lora_adapter(model, path)
```

原因：

```text
普通 MiniGPT 没有 lora_A/lora_B 这些 key。
只有先接入 LoRA，adapter checkpoint 里的权重才有地方加载。
```

## 面试积累

### Q1: LoRA adapter checkpoint 里应该保存什么？

回答草稿：

```text
只保存 LoRA 新增并训练过的低秩矩阵，例如 lora_A.weight 和 lora_B.weight。
base model 的原始 weight/bias 不属于 adapter，应该由 base checkpoint 单独提供。
```

### Q2: 为什么不能把 LoRA adapter 直接加载到普通 MiniGPT？

回答草稿：

```text
普通 MiniGPT 的模块结构里没有 lora_A/lora_B。
必须先把目标 Linear 替换成 LoRALinear，模型 state_dict 里才有对应 key，adapter 权重才能加载进去。
```

### Q3: `strict=False` 在加载 adapter 时解决了什么？

回答草稿：

```text
adapter checkpoint 只有 LoRA 权重，缺少 embedding、FFN、head、base qkv 等大量 key。
strict=False 允许 checkpoint 只提供部分 key，从而只更新模型中的 LoRA 参数。
```

## 今日误区

```text
误区：保存 LoRA adapter 也应该按 requires_grad 筛选。
修正：optimizer 筛选训练参数看 requires_grad；adapter checkpoint 更稳的做法是按 state_dict 的 key 名筛选 lora_。
```

```text
误区：strict=False 会自动创建 LoRA 层。
修正：strict=False 只允许部分 key 缺失，不会改变模型结构。加载前必须先接入 LoRA。
```
