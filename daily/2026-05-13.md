## 参数流推导

### optimizer 参数筛选

```python
trainable_params = [
    p
    for p in model.parameters()
    if p.requires_grad
]
optimizer = torch.optim.AdamW(trainable_params, lr=3e-4)
```

含义：

```text
requires_grad=False 的 frozen base 参数仍然用于 forward，
但不会累积梯度，也不会被 optimizer 更新。
```

### cross entropy shape

如果：

```text
logits:  (2, 8, 50)
targets: (2, 8)
```

那么：

```python
loss = F.cross_entropy(
    logits.view(-1, logits.size(-1)),
    targets.view(-1),
)
```

对应：

```text
input:  (16, 50)
target: (16,)
```

## 面试积累

### Q1: LoRA 训练时为什么 optimizer 只放可训练参数？

回答草稿：

```text
因为 LoRA 的目标是冻结 base model，只训练新增的低秩 adapter。
把 optimizer 的参数列表筛成 requires_grad=True，可以避免把 frozen 参数放进优化器状态里，
也能更清楚地确认真正更新的是 LoRA A/B。
```

### Q2: frozen 参数还会参与 forward 吗？

回答草稿：

```text
会。冻结只表示不训练、不更新，不表示 forward 不使用。
LoRA 的输出仍然是 base_out + lora_out，其中 base_out 来自冻结的原始 Linear。
```

### Q3: 为什么用 `lora_B.weight` 检查训练一步后 adapter 是否变化？

回答草稿：

```text
因为当前实现里 lora_A 随机初始化，lora_B 初始化为 0。
第一步 backward 时，lora_B 通常会直接收到有效梯度并更新；
lora_A 的梯度可能因为 lora_B 初始为 0 而暂时为 0。
```

## 今日误区

```text
误区：optimizer 直接传 model.parameters() 和筛选 trainable 参数完全一样。
修正：如果 frozen 参数没有梯度，通常不会被更新；但 LoRA 训练里更推荐显式筛选 requires_grad=True，
这样 optimizer 状态更小，训练对象更清楚。
```

```text
误区：targets.view(-1) 会变成 (16, 1)。
修正：view(-1) 会拉平成一维，所以 (2, 8) 会变成 (16,)。
```
