## 40 分钟学习

- 来源：`projects/mini-gpt-from-scratch/sample.py`
- 概念：logits、softmax、multinomial、temperature、top-k sampling、top-p sampling
- 用自己的话解释：
  - `sample.py` 生成时只取 `logits[:, -1, :]`，因为最后一个位置代表基于完整当前上下文预测下一个 token 的分布。
  - temperature 不改变模型参数，只是在 softmax 前缩放 logits，改变概率分布的尖锐程度。
  - temperature 小于 1 时，高分 token 和低分 token 的差距被放大，生成更稳定但更容易重复。
  - temperature 大于 1 时，分布变平，低概率 token 更容易被抽到，生成更随机但更容易跑偏。
  - top-k 固定保留分数最高的 k 个候选 token，其他 token 被设为 `-inf`，softmax 后概率为 0。
  - top-p 不固定 token 个数，而是从高概率 token 开始累加，保留累计概率达到 p 的候选集合。
- 仍然不懂的地方：
  - 需要继续多跑不同 prompt，观察 top-p 和 top-k 在更不确定分布下的差异。


- 学到了什么：

  - top-p 会根据当前概率分布动态决定保留多少 token。

## 面试积累

今天的工作能帮助我回答哪个问题？

### Q1: temperature 改变了什么？

回答草稿：

```text
temperature 会在 softmax 前缩放 logits。temperature 越小，分布越尖锐，高分 token 概率越高；temperature 越大，分布越平，低概率 token 更容易被采样到。
```

### Q2: top-k 和 temperature 有什么区别？

回答草稿：

```text
temperature 改变整个概率分布的形状，所有 token 仍然可能被采样；top-k 直接限制候选范围，只保留分数最高的 k 个 token，其余 token 概率变成 0。
```

### Q3: 采样策略会让模型变聪明吗？

回答草稿：

```text
不会。采样策略不改变模型参数，只改变从 logits 分布中选择下一个 token 的方式。它能影响稳定性、多样性和重复程度，但不会提高模型本身学到的知识。
```

### Q4: top-k 和 top-p 有什么区别？

回答草稿：

```text
top-k 固定保留概率最高的 k 个 token；top-p 固定保留累计概率达到 p 的候选集合，所以保留的 token 个数会随着当前分布动态变化。
```

## 今日一句话

```text
temperature 控制随机性，top-k 固定候选数量，top-p 固定累计概率质量；采样策略调的是生成风格，不是模型能力。
```
