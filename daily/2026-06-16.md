# 2026-06-16 - Day34 3B QLoRA 过渡实验

## 今天为什么先做 3B

昨天 `0.5B + 十几条 LoRA 教学样本` 在 50 题固定评测上不稳定。今天没有直接上 7B，而是先用 `Qwen/Qwen2.5-3B-Instruct` 做过渡实验。

这个实验想回答的问题是：

```text
0.5B 的失败，到底主要是模型太小，还是数据/训练目标本身不够好？
```

如果 3B 明显变好，说明模型规模确实有帮助，后面上 7B 就更有意义。如果 3B 也不稳，就说明不能只靠换大模型，还要继续改数据和评测。

今天继续使用同一份 50 题固定评测集：


先跑了 3B base baseline：

```text

```

然后跑了 base + adapter 对照：

```text
runs/day34_3b_base_vs_adapter_eval.jsonl
runs/day34_3b_base_vs_adapter_eval.md
100 rows = 50 base + 50 adapter
```

前 10 条 `concept_*` 人工粗看：

```text
3B adapter 明显学到了“先说结论”的短答格式，
但概念仍然没有完全稳定。
```

比较好的回答包括：

- LoRA 不是 pretraining
- LoRA 不更新全部 base weights
- LoRA 是低秩增量
- QLoRA 不量化正在训练的 adapter

仍然错误或不稳的地方包括：

- 把 optimizer 更新对象说成 `LoRA adapter 和 base weights`
- 把 `lora_alpha` 说成量化超参
- 把 QLoRA 省显存说成只保留 base weights / 只训练 adapter，而没有说清 4bit 量化 frozen base weights
- 把 LoRA 和 QLoRA 可训练参数一样解释成 adapter 量化
- 对 quantized base forward 的解释仍然混乱

## 今天的结论

3B 比 0.5B 更像在认真回答，也更容易学到固定回答风格，但它仍然不是稳定的 LoRA/QLoRA 概念老师。

今天最重要的一句话：

```text
3B QLoRA 证明了链路能放大到更真实的模型，但也证明了“模型变大”不能自动修好小数据里的概念错误。
```

下一步不应该只看 loss，也不应该盲目上 7B。更合理的是：

1. 继续保留 3B/7B 这条真实模型链路。
2. 把 `lora_alpha`、QLoRA 省显存、trainable params、quantized forward 这些错点补成更明确的数据。
3. 用同一份 50 题评测验证下一轮 adapter。
4. 如果 3B 的修正数据有效，再上 7B。
