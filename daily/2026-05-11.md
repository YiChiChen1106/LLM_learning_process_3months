## 40 分钟学习

- 来源：`projects/toy-lora-from-scratch/toy_lora_linear.py`
- 概念：`nn.Linear` 参数量、冻结原始 `W`、低秩矩阵 `A/B`、`rank`、`alpha / r`、`total parameters`、`trainable parameters`
- 用自己的话解释：
  - 普通 `nn.Linear(in_features, out_features)` 的权重 shape 是 `(out_features, in_features)`。
  - full fine-tuning 会直接训练原始 Linear 的 `weight`，通常也包括 `bias`。
  - LoRA 冻结原始 Linear，只训练额外加上的低秩矩阵 `A` 和 `B`。
  - LoRA 输出不是丢掉原始 `W`，而是 `base_out + lora_out`。
  - LoRA 的 `total parameters` 会因为新增 adapter 变多，但 `trainable parameters` 会大幅变少。

## 参数量推导

### 普通 Linear

```text
nn.Linear(128, 384, bias=True)

weight: (384, 128) => 384 * 128 = 49152
bias:   (384,)     => 384
total/trainable    => 49536
```

### LoRALinear

```text
LoRALinear(128, 384, r=8, bias=True)

base weight: (384, 128) => 49152，冻结
base bias:   (384,)     => 384，冻结
A:           (8, 128)   => 1024，可训练
B:           (384, 8)   => 3072，可训练

total parameters     = 49152 + 384 + 1024 + 3072 = 53632
trainable parameters = 1024 + 3072 = 4096
```

## 面试积累

### Q1: 普通 `nn.Linear(128, 384, bias=True)` 有多少参数？

回答草稿：

```text
PyTorch 的 Linear 权重 shape 是 (out_features, in_features)，所以 weight 是 (384, 128)，bias 是 (384,)。
总参数量是 384 * 128 + 384 = 49536。
```

### Q2: LoRA 冻结原始 `W`，为什么模型输出仍然可以改变？

回答草稿：

```text
LoRA 冻结原始 W，但额外训练一个低秩更新分支。最终输出是 base_out + lora_out，而不是只看原始 W，所以模型行为仍然可以改变。
```

### Q3: LoRA 的 `A` 和 `B` shape 怎么定？

回答草稿：

```text
如果原始 Linear 是 in_features -> out_features，LoRA rank 是 r，那么 A 是 (r, in_features)，B 是 (out_features, r)。代码里可以写成 Linear(in_features, r) 和 Linear(r, out_features)。
```

### Q4: 为什么 LoRA 常把 `B` 初始化为 0？

回答草稿：

```text
B 初始化为 0 时，lora_out 一开始就是 0，所以 LoRALinear 初始输出等于原始 Linear 输出。这样训练开始时不会突然扰乱原模型，只是在训练中逐渐学出修正项。
```

### Q5: LoRA 的 `total parameters` 会更少吗？

回答草稿：

```text
通常不会。LoRA 保留原始 Linear，又额外增加 A/B，所以 total parameters 会更多。LoRA 主要减少的是 trainable parameters、梯度和优化器状态。
```

## 今日误区

```text
误区：full fine-tuning 只训练 W，不用管 bias。
修正：如果 bias=True，full fine-tuning 通常也会训练 bias；算参数量时不要漏掉 bias。
```

```text
误区：LoRA 让模型总参数量变少。
修正：LoRA 通常会增加少量 adapter 参数；真正变少的是可训练参数量。
```

```text
误区：冻结 W 就代表 forward 不再使用 W。
修正：冻结只是不更新参数，forward 仍然要用原始 W 计算 base_out。
```

## 今日一句话

```text
LoRA 不重写原来的大 W，而是在旁边训练一个小的低秩修正项；总参数可能更多，但需要更新的参数少很多。
```
