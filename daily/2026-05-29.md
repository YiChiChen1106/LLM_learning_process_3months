## 看到的关键结果

```text
full fine-tuning trainable = 128 * 384 + 384 = 49,536
LoRA / QLoRA trainable     = (128 + 384) * 8 = 4,096

base.weight params = 128 * 384 = 49,152
fp16 storage       = 49,152 * 2 = 98,304 bytes
4-bit storage      = 49,152 * 0.5 = 24,576 bytes
```

## 我能解释的概念

### 1. QuantizedLinear

`QuantizedLinear` 把普通 `nn.Linear` 的 float weight 量化成 `qweight + scale`。`qweight`、`scale` 和 `bias` 是 buffer，会进入 `state_dict()`，但不会进入 `model.parameters()`。

### 2. QLoRALinear

`QLoRALinear` 的 forward 是：

```python
out = quantized_base(x) + scaling * lora_B(lora_A(x))
```

其中 base 是 frozen quantized linear，LoRA adapter 是可训练的 `lora_A.weight` 和 `lora_B.weight`。

### 3. merge_qlora_linear

merge 时要先反量化 base weight，再加 LoRA delta：

```python
base_weight_hat = qweight.float() * scale
delta_weight = lora_B.weight @ lora_A.weight
merged_weight = base_weight_hat + scaling * delta_weight
```

merge 后变成普通 `nn.Linear`，`state_dict()` 里只剩 `weight` 和 `bias`，适合推理，不适合继续 QLoRA adapter 训练。

### 4. state_dict 和 parameters

`state_dict()` 用于保存和加载，会包含 Parameter 和 buffer。`model.parameters()` 用于训练和 optimizer，只包含 Parameter。

所以 toy `QLoRALinear.state_dict()` 包含：

```text
base.qweight
base.scale
base.bias
lora_A.weight
lora_B.weight
```

但 `QLoRALinear.parameters()` 只包含：

```text
lora_A.weight
lora_B.weight
```

### 5. LoRA 和 QLoRA 的 trainable parameters

LoRA 和 QLoRA 通常训练的是同一组 adapter 参数，所以 trainable parameters 一样。区别主要在 frozen base weight 的存储方式：

```text
LoRA:  base weight 用 fp16 / bf16 存
QLoRA: base weight 用低比特量化形式存
```

## 今天的一个误区

`qweight` 不能直接和 LoRA delta 相加。`qweight` 是整数编码，必须先用 `scale` 反量化成近似 float weight。

## 今天的结论

QLoRA 相比 LoRA，多省的是 frozen base weights 的存储显存；没有改变 LoRA adapter 的可训练参数量，也没有改变只训练 adapter 的训练方式。
