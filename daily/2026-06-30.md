## 自动判分规则补齐

今天检查发现：

所以后面把 50 题的自动判分规则补齐了。

```text
questions=50

```

这意味着现在 50 题固定评测集基本不会再大面积出现 `ungraded`。

## 判分规则的思路

今天的判分器仍然不是大模型判分，而是透明的 rule-based 判分。

每条规则大致包含：

```text
required: 必须命中的核心要点
forbidden: 明显错误说法
feedback_correct: 答对时反馈
review_tip: 复习建议
```

如果命中全部 required，判为 `correct`。

如果命中部分 required，判为 `partial`。

如果命中 forbidden，判为 `wrong`。

这个方式的优点是透明、可控、容易用真实作答不断校准。

## 修复误判

今天用真实 quiz 回答校准了不少误判。

例如：

```text
不是因为 adapter 参数更少
```

这句话是在否定错误观点，但旧规则只看到 `adapter 参数更少`，会误判成 wrong。

所以今天加了否定上下文识别：

```text
不是因为
并不是因为
不是靠
不靠
不算
不能算
不能
不是
并不是
不
```

这样类似：

```text
不是因为 adapter 参数更少
不算完全正确
不能完全替代强模型
```

不会再被里面的错误关键词误伤。

## 拼写和输入法容错

真实回答里会有输入法错误或英文中文混写。

今天补了一些容错词，例如：

```text
参数高校 -> 参数高效
frozen wight -> frozen weight
不属于pretraining
base weight不更新
base weight是frozen
```

这让判分器更适合真实使用，而不是只能匹配标准答案里的固定句子。

## 推荐原因细分

今天还发现推荐原因太粗：

```text
优先练习薄弱类别: concept。
```

这个粒度太大，容易一直在 `concept` 里打转，看不出到底薄弱在哪里。

所以新增了 `prompt_subtopic()`，把 50 题映射到更细的知识点：

```text
LoRA 基础
QLoRA 量化
adapter 与 merge
target_modules
Dataset / tokenizer
labels 与 loss
padding / collator
训练配置
评测闭环
失败模式
模型能力边界
数据来源与质量
```

现在推荐原因会更像：

```text

优先练习薄弱知识点: labels 与 loss。
优先练习薄弱知识点: adapter 与 merge。
```

而不是只说 `concept` 或 `dataflow`。

## 今天重点复习到的概念

### LoRA 和 QLoRA

LoRA 和 QLoRA 的可训练参数数量通常一样，因为训练对象仍然是 adapter。

区别主要在于：

```text
LoRA: base weights 通常冻结，但以普通精度加载
QLoRA: frozen base weights 用 4bit 量化加载
```

所以 QLoRA 省的是 frozen base weights 的存储显存，不是把 adapter 参数变少。

### adapter 不能单独 generate

`adapter_model.safetensors` 不是完整模型。

它通常只包含：

```text
lora_A
lora_B
adapter config
```

如果没有对应 base model，不能直接聊天或 generate。

### target_modules

`target_modules` 决定哪些模块会被加 LoRA adapter。

例如：

```text
q_proj
k_proj
v_proj
o_proj
```

对应 attention 里的 query/key/value/output projection。

如果名字写错，PEFT 可能找不到目标模块，导致报错或没有正确插入 LoRA。

### labels 和 loss

`attention_mask` 和 `labels=-100` 不是同一件事。

```text
attention_mask: 管哪些 token 是有效输入，影响 attention
labels=-100: 管哪些位置不参与 loss
```

这也是今天规则里重点覆盖的一组知识点。
